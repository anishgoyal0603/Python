# 05. Style Guide

## What is a Style Guide?
* **Definition:** A set of conventions on how to format and write code properly. While Python allows many ways to write things, following a style guide ensures consistency.
* **Industry Standard:** Large MNCs have strict coding standards. Beginners are recommended to follow the **Google Python Style Guide**.

## Naming Cases
1. **Snake Case:** `total_price` (all lowercase, words separated by underscores).
2. **Camel Case:** `totalPrice` (first word lowercase, subsequent words start with a capital letter).
3. **Pascal Case:** `TotalPrice` (all words start with a capital letter).

## Python's Preferred Style
* **Recommendation:** Python strongly prefers **Snake Case** for variables and functions.
* **Reasoning:** Built-in Python libraries and existing functions use snake case. Matching this style makes your code consistent with the language itself.

## Example Program: Sum of Two Numbers
* Create two variables and assign values: `a = 5`, `b = 10`
* Add them using a third variable: `sum = a + b`
* Output the result: `print(sum)`
* Running this code yields the answer: `15`.
